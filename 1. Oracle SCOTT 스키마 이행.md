
### 4. PostgreSQL Perl 드라이버 설치 (DBD::Pg)

Oracle Linux 7 환경에서 발생하는 주요 이슈는 아래와 같다.

| 이슈 | 원인 | 해결 방법 |
|------|------|---------|
| postgresql-devel 버전 충돌 | 기본 yum 레포의 postgresql-devel은 9.2 버전 | PGDG의 postgresql15-devel 사용 |
| llvm-toolset-7-clang 없음 | Oracle Linux 7 SCL 레포에 패키지 없음 | --nodeps로 헤더만 설치 |
| ExtUtils::MakeMaker 버전 부족 | Oracle Linux 7 perl은 6.68 | local::lib으로 업그레이드 |

#### 필요한 저장소 활성화

```bash
yum install -y oracle-epel-release-el7
yum install -y oracle-softwarecollection-release-el7
yum-config-manager --enable ol7_software_collections
yum-config-manager --enable ol7_optional_latest

# 확인
yum repolist | grep -iE "epel|software|optional"
```

#### 보조 devel 패키지 설치

```bash
yum install -y perl-IPC-Run libedit-devel libicu-devel
```

#### postgresql15-devel 헤더 설치 (--nodeps)

```bash
cd /tmp
yumdownloader postgresql15-devel
rpm -ivh --nodeps postgresql15-devel-*.rpm

# 헤더 확인
ls /usr/pgsql-15/include/libpq-fe.h
/usr/pgsql-15/bin/pg_config --includedir
/usr/pgsql-15/bin/pg_config --libdir
```

#### ExtUtils::MakeMaker 업그레이드

```bash
cpan ExtUtils::MakeMaker
```

> yes → local::lib → yes → yes 순서로 입력한다.
> 설치 완료 후 ~/.bashrc에 환경변수 5줄이 자동 추가된다.

```bash
source ~/.bashrc
perl -MExtUtils::MakeMaker -e 'print "$ExtUtils::MakeMaker::VERSION\n"'
# 7.64 이상이면 정상
```

#### DBD::Pg 설치

```bash
POSTGRES_HOME=/usr/pgsql-15 cpan DBD::Pg
```

```bash
perl -MDBD::Pg -e 'print "DBD::Pg $DBD::Pg::VERSION OK\n"'
```

### 8. Ora2Pg 프로젝트 생성 및 설정

```bash
su - oracle
mkdir -p ~/ora2pg_lab
cd ~/ora2pg_lab

ora2pg --project_base . --init_project scott_to_pg
cd scott_to_pg
```

ora2pg.conf 설정을 반영한다.

```bash
sed -i \
  -e 's|^ORACLE_HOME.*|ORACLE_HOME /u01/app/oracle/product/19.3.0/dbhome_1|' \
  -e 's|^ORACLE_DSN.*|ORACLE_DSN dbi:Oracle:host=192.168.23.67;sid=orcl;port=1521|' \
  -e 's|^ORACLE_USER.*|ORACLE_USER scott|' \
  -e 's|^ORACLE_PWD.*|ORACLE_PWD tiger|' \
  -e 's|^SCHEMA.*|SCHEMA SCOTT|' \
  -e 's|^PG_SCHEMA.*|PG_SCHEMA scott|' \
  -e 's|^#NLS_LANG.*|NLS_LANG AMERICAN_AMERICA.AL32UTF8|' \
  -e 's|^#CLIENT_ENCODING.*|CLIENT_ENCODING UTF8|' \
  config/ora2pg.conf

grep -E '^ORACLE_|^SCHEMA|^PG_SCHEMA|^NLS_LANG|^CLIENT_ENCODING' config/ora2pg.conf
```

### 9. 이관 사전 평가

```bash
# Oracle 접속 확인
ora2pg -t SHOW_VERSION -c config/ora2pg.conf

# 테이블 목록 확인
ora2pg -t SHOW_TABLE -c config/ora2pg.conf

# 이관 난이도 리포트 생성
ora2pg -t SHOW_REPORT -c config/ora2pg.conf --estimate_cost > reports/migration_report.txt
cat reports/migration_report.txt
```

이관 난이도 등급 기준은 아래와 같다.

| 등급 | 설명 |
|------|------|
| A | 도구에 의해 자동 이관 가능 |
| B | 코드 재작성 필요 / 5일 이하 |
| C | 코드 재작성 필요 / 5일 초과 |
| 1 | 저장 함수/트리거 없음 (단순) |
| 2 | 트리거 있음 / 수동 재작성 불필요 |
| 3 | 저장 함수/트리거 있음 / 수동 재작성 불필요 |

### 10. 스키마 (DDL) 추출

```bash
ora2pg -t TABLE    -o tables.sql    -b schema/tables    -c config/ora2pg.conf
ora2pg -t SEQUENCE -o sequences.sql -b schema/sequences -c config/ora2pg.conf
ora2pg -t INDEX    -o indexes.sql   -b schema/indexes   -c config/ora2pg.conf
ora2pg -t VIEW     -o views.sql     -b schema/views     -c config/ora2pg.conf
ora2pg -t GRANT    -o grants.sql    -b schema/grants    -c config/ora2pg.conf

cat schema/tables/tables.sql
```

### 13. 이관 검증

Oracle과 PostgreSQL의 테이블 건수를 비교한다.

Oracle:

```bash
sqlplus -S scott/tiger@192.168.23.67:1521/orcl <<EOF
SELECT 'EMP'      AS tbl, COUNT(*) cnt FROM emp
UNION ALL SELECT 'DEPT',     COUNT(*) FROM dept
UNION ALL SELECT 'SALGRADE', COUNT(*) FROM salgrade
UNION ALL SELECT 'BONUS',    COUNT(*) FROM bonus
ORDER BY 1;
EXIT
EOF
```

PostgreSQL:

```bash
psql -c "
SELECT 'EMP'      AS tbl, COUNT(*) FROM scott.emp
UNION ALL SELECT 'DEPT',     COUNT(*) FROM scott.dept
UNION ALL SELECT 'SALGRADE', COUNT(*) FROM scott.salgrade
UNION ALL SELECT 'BONUS',    COUNT(*) FROM scott.bonus
ORDER BY 1;
"
```

> 양쪽 건수가 완전히 일치해야 한다.
