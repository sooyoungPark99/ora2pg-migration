# Oracle HR 스키마 이행

## SCOTT과 HR의 차이점

| 오브젝트 | SCOTT | HR |
|---------|-------|-----|
| 시퀀스 | 없음 | 3개 (DEPARTMENTS_SEQ, EMPLOYEES_SEQ, LOCATIONS_SEQ) |
| 트리거 | 없음 | 2개 (SECURE_EMPLOYEES, UPDATE_JOB_HISTORY) |
| CHECK 제약 | 없음 | 있음 (JOB_HISTORY end>start, EMPLOYEES salary>0) |
| Self FK | 없음 | EMPLOYEES.MANAGER_ID → EMPLOYEES |

> HR은 import 순서, PL/SQL 수동 보정, 시퀀스 setval이 SCOTT보다 훨씬 중요하다.

### 4. ora2pg.conf 설정

```bash
sed -i -e 's|^ORACLE_HOME.*|ORACLE_HOME /u01/app/oracle/product/19.3.0/dbhome_1|' config/ora2pg.conf
sed -i -e 's|^ORACLE_DSN.*|ORACLE_DSN dbi:Oracle:host=192.168.13.181;sid=orcl;port=1521|' config/ora2pg.conf
sed -i -e 's|^ORACLE_USER.*|ORACLE_USER hr|' config/ora2pg.conf
sed -i -e 's|^ORACLE_PWD.*|ORACLE_PWD hr|' config/ora2pg.conf
sed -i -e 's|^SCHEMA.*|SCHEMA HR|' config/ora2pg.conf
sed -i -e 's|^PG_SCHEMA.*|PG_SCHEMA hr|' config/ora2pg.conf
sed -i -e 's|^#NLS_LANG.*|NLS_LANG AMERICAN_AMERICA.AL32UTF8|' config/ora2pg.conf
sed -i -e 's|^#CLIENT_ENCODING.*|CLIENT_ENCODING UTF8|' config/ora2pg.conf

grep -E '^ORACLE_|^SCHEMA|^PG_SCHEMA|^NLS_LANG|^CLIENT_ENCODING' config/ora2pg.conf
```

> HR처럼 시퀀스/트리거가 있는 경우 아래 옵션도 필요에 따라 조정한다.
> - FKEY_DEFERRABLE 1 : FK를 DEFERRABLE로 내보냄
> - DEFER_FKEY 1 : import 트랜잭션에서 FK 검사를 지연 (순환 FK에 유용)

### 8. PostgreSQL로 Import

import 순서가 중요하다. FK는 반드시 데이터 적재 이후에 실행한다.

```bash
cd ~/ora2pg_lab/hr_to_pg

export PGHOST=localhost
export PGPORT=5432
export PGDATABASE=testdb
```

#### 1. 테이블 구조 Import

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/tables/tables.sql
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/tables/CONSTRAINTS_tables.sql
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/tables/INDEXES_tables.sql
```

> FKEYS_tables.sql은 데이터 적재 완료 후 실행한다.

#### 2. 시퀀스 Import

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/sequences/sequences.sql
```

#### 3. 데이터 Import

HR은 DEPARTMENTS ↔ EMPLOYEES 간 순환 FK가 있어 session_replication_role = replica 설정이 필요하다.

```bash
PGUSER=postgres PGPASSWORD=oracle psql -v ON_ERROR_STOP=1
```

```sql
SET session_replication_role = replica;
\i data/data.sql
SET session_replication_role = origin;
\q
```

> replica 모드는 FK와 트리거를 일시 비활성화하여 순환 FK 문제를 해결한다.
> CHECK / NOT NULL은 유지되므로 데이터 자체가 CHECK를 위반하면 오류가 발생한다.

#### 4. FK Import (데이터 적재 직후)

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/tables/FKEYS_tables.sql
```

#### 5. 뷰 Import

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/views/views.sql
```

#### 6. 프로시저 Import

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/procedures/procedures.sql
```

#### 7. 트리거 Import (마지막)

트리거는 데이터 적재를 방해할 수 있으므로 가장 마지막에 실행한다.

```bash
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/triggers/triggers.sql
```

#### 8. 권한 Import

```bash
cat schema/grants/grants.sql
PGUSER=hr PGPASSWORD=hr psql -v ON_ERROR_STOP=1 -f schema/grants/grants.sql
```

### 9. 시퀀스 Last Value 동기화

HR 이관 후 반드시 수행해야 한다. 시퀀스가 낮게 잡혀 있으면 INSERT 시 PK 중복이 발생한다.

```bash
PGPASSWORD=hr psql -h localhost -U hr -d testdb
```

```sql
SELECT setval('hr.employees_seq',   (SELECT COALESCE(MAX(employee_id),   0) + 1 FROM hr.employees),   false);
SELECT setval('hr.departments_seq', (SELECT COALESCE(MAX(department_id), 0) + 1 FROM hr.departments), false);
SELECT setval('hr.locations_seq',   (SELECT COALESCE(MAX(location_id),   0) + 1 FROM hr.locations),   false);
```

동기화 결과를 확인한다.

```sql
SELECT 'employees_seq'   AS seq, last_value, is_called FROM hr.employees_seq;
SELECT 'departments_seq' AS seq, last_value, is_called FROM hr.departments_seq;
SELECT 'locations_seq'   AS seq, last_value, is_called FROM hr.locations_seq;
\q
```

> setval(..., false)의 세 번째 인자 false는 "다음 nextval 호출에서 이 값을 그대로 반환"을 의미한다.
