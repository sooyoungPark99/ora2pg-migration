# DBeaver로 PostgreSQL 접속

### 1. 서버에서 hr 접속 확인

서버에서 hr 유저로 psql에 접속하여 이행된 오브젝트를 확인한다.

```bash
psql -U hr -d testdb
```

```sql
-- 테이블 목록
\dt

-- 인덱스 목록
\di

\q
```

```bash
# 서버 IP 확인
ifconfig
```

---

### 2. DBeaver 새 연결 생성

1. DBeaver 실행
2. 상단 메뉴 파일 → 새로 만들기 또는 플러그 아이콘(+) 클릭
3. Database Connection 선택 → PostgreSQL 선택 → 다음

---

### 3. 연결 정보 입력

| 항목 | 값 |
|------|-----|
| Host | 192.168.23.67 |
| Port | 5432 |
| Database | testdb |
| Username | hr |
| Password | hr |

> Database 이름은 hr 스키마가 어느 DB에 있느냐에 따라 다를 수 있다.
> 모르면 postgres로 먼저 시도한다.

---

### 4. 드라이버 설치 (최초 1회)

PostgreSQL 드라이버가 없으면 자동으로 드라이버 다운로드 팝업이 뜬다.
Download 버튼을 클릭하면 자동 설치된다.

---

### 5. 연결 테스트 후 완료

1. Test Connection 클릭 → Connected 메시지 확인
2. 완료(Finish) 클릭

### 6. 접속 후 확인용 SQL

```sql
-- 테이블 목록
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_type = 'BASE TABLE'
  AND table_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY table_schema, table_name;

-- 인덱스 목록
SELECT schemaname, tablename, indexname, indexdef
FROM pg_indexes
WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
ORDER BY schemaname, tablename, indexname;

-- 제약 조건 목록
SELECT constraint_schema, table_name, constraint_name, constraint_type
FROM information_schema.table_constraints
WHERE constraint_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY constraint_schema, table_name, constraint_name;

-- 뷰 목록
SELECT table_schema, table_name
FROM information_schema.views
WHERE table_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY table_schema, table_name;

-- 시퀀스 목록
SELECT sequence_schema, sequence_name, data_type,
       start_value, minimum_value, maximum_value, increment
FROM information_schema.sequences
WHERE sequence_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY sequence_schema, sequence_name;

-- 테이블 권한
SELECT table_schema, table_name, grantee, privilege_type
FROM information_schema.table_privileges
WHERE table_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY table_schema, table_name, grantee, privilege_type;
```
